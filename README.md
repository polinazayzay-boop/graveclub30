Обе использовались одинаково: я формулировала задачу текстом, получала готовый фрагмент кода, вставляла в проект и при необходимости уточняла правки.

Что сделано через нейросети:

Редактируемое имя

Задача: по клику на текст появлялось поле ввода, после ввода текст заменялся.
Процесс: нейросети предложили замену <p> на <input> с фокусом и заменой обратно (в коде осталась только первая часть, так как дальнейшее доделывалось отдельно).

Кнопки оплаты (pay1, pay2)

Задача: при клике на одну кнопку она подсвечивается, другая снимает подсветку, и снизу выводится разный текст.
Процесс: нейросети написали функцию сброса классов active и установку сообщения в зависимости от нажатой кнопки.

Рисовалка (canvas)

Задача: рисование мышкой и пальцем.
Процесс: нейросети добавили обработчики событий для мыши (mousedown, mousemove, mouseup) и для тач-экранов (touchstart, touchmove, touchend), а также функцию приведения координат.

Переключатель артистов (3 кнопки, 3 попапа)

Задача: по умолчанию активна первая кнопка и показан её попап; при клике на другую кнопку — меняется активный класс и показывается соответствующий попап.
Процесс: нейросети создали функции resetArtistActive() и hideArtistPopups(), повесили обработчики.

Музыкальные плееры (Lana, Tame, Tyler, Bunny)

Задача:

клик по картинке — запуск трека, остальные останавливаются;
при повторном клике трек останавливается;
во время проигрывания картинка меняется, после окончания — возвращается;
для одного трека показывается сообщение best, для остальных — others.
Процесс: нейросети написали функцию stopAllExcept(), которая останавливает все треки, кроме текущего, и обработчики для каждой картинки с проверкой состояния воспроизведения, сменой картинки и показом сообщения.

Игра «Угадай число»

Задача: загадывается число от 1 до 10, пользователь вводит вариант, выводится подсказка «Больше», «Меньше» или «Угадал!», после угадывания число меняется, поле очищается.
Процесс: нейросети написали проверку ввода, сравнение с загаданным, генерацию нового числа после победы.

Игра «Мемори» 

Задача: 8 карточек с классами c1–c8, пары: (c1,c2), (c3,c4), (c5,c6), (c7,c8). При клике карточка открывается; если пара совпадает — помечается классом done и остаётся открытой; если нет — через 0.9 сек закрывается. После открытия всех пар показывается сообщение о победе.
Процесс: нейросети написали логику с переменными first и lock, проверку на пару по комбинациям классов, таймаут для несовпавших, и функцию checkWin().

Блок «Придёшь?» (кнопки Да/Нет)

Задача:

кнопка «Нет» при каждом нажатии уменьшается, кнопка «Да» — увеличивается;
когда «Нет» становится слишком маленькой — исчезает и становится неактивной;
при нажатии «Да» вопрос заменяется на ответ, обе кнопки плавно исчезают.
Процесс: нейросети написали код с масштабированием (transform: scale), проверкой состояния, и финальным скрытием.
